# Sottoinsieme di requirements.txt: solo cio' che serve a
# `streamlit run dashboard/Home.py` per leggere lo snapshot statico in
# `data/dashboard_export/` (vedi wiki/pages/dashboard.md) - nessuna
# connessione a Postgres/PostGIS. Pensato per Streamlit Community Cloud, dove
# dipendenze pesanti con librerie di sistema (GDAL/rasterio, HDF5/netCDF4,
# CDS API) rischiano di rallentare o far fallire l'installazione e non sono
# comunque mai usate dal codice della dashboard.
#
# `sqlalchemy` resta necessaria anche qui: `dashboard/components/queries.py`
# non parla piu' con il DB, ma importa a runtime alcune funzioni pure di
# `src/analysis/trend_analysis.py`/`seasonal_analysis.py`/`spatial_analysis.py`,
# che a livello di modulo fanno `from sqlalchemy import text` e
# `from src.utils.database import db_manager` (mai una connessione vera,
# solo l'istanza lazy di `DatabaseManager` - l'engine si crea solo alla prima
# query, mai eseguita da questi percorsi). `psycopg2-binary`/`geoalchemy2`
# invece non servono: nessun codice raggiungibile dalla dashboard crea
# davvero un engine Postgres.
